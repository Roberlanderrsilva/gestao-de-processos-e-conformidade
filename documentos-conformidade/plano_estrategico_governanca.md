# 🛡️ Plano de Ação e Governança GRC


Este documento define as contramedidas e ações estratégicas para manter a conformidade da infraestrutura de TI e a proteção de dados (LGPD).

### 📝 1. Identificação e Diagnóstico
Todos os riscos listados na **Matriz de Riscos** são monitorados semanalmente. O script Python de auditoria identifica falhas de segurança e inconformidades contratuais.

### 🚀 2. Plano de Resposta (Ações Práticas)
* **Controle de Acesso:** Revisão trimestral de usuários e permissões de sistema.
* **Segurança de Dados:** Implementação de rotinas de criptografia em bancos de dados com dados pessoais.
* **Treinamento:** Workshop semestral para a equipe sobre boas práticas de segurança da informação.

### 📊 3. Cronograma de Auditoria
* **Janeiro/Julho:** Auditoria completa de software e licenciamento.
* **Contínuo:** Monitoramento automatizado via terminal (Python) para alertas de risco crítico.
